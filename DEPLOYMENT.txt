Employee Attendance & Payroll — deployment

Login (Angular + API)
- POST `/api/auth/login` with `{ "username": "admin", "password": "admin" }` (defaults in `appsettings.json` under `Auth`). Change `Auth:Username` and `Auth:Password` for production.
- The SPA stores the JWT in `sessionStorage` and sends `Authorization: Bearer …` on API calls. Open the site, click Login on the landing page, then use the app shell.

Prerequisites
- .NET 10 SDK
- Node.js 20+ (for Angular CLI / npm)
- Microsoft SQL Server (any edition you use with SQL Server Management Studio), OR SQL Server Express LocalDB (installed with Visual Studio / Build Tools)

Database (SQL Server — SSMS)

1) Connection string
- Default in appsettings.json targets LocalDB (good for dev + SSMS on the same PC):
  Server=(localdb)\MSSQLLocalDB;Database=EmployeePayroll;Trusted_Connection=True;MultipleActiveResultSets=true;TrustServerCertificate=True
- Put this under ConnectionStrings:SqlServer (already set). You can override in appsettings.Development.json or user secrets.

2) SQL Server Express (named instance) example
- If you use SSMS against .\SQLEXPRESS, set SqlServer to:
  Server=.\SQLEXPRESS;Database=EmployeePayroll;Trusted_Connection=True;MultipleActiveResultSets=true;TrustServerCertificate=True

3) View data in SSMS
- Open SQL Server Management Studio.
- Connect to the same server as in your connection string, e.g. (localdb)\MSSQLLocalDB or .\SQLEXPRESS.
- Expand Databases → EmployeePayroll → Tables. You should see Employees and Payrolls after the API has run once (migrations create the schema).

4) Optional: apply schema without running the API
- Run the generated script in SSMS against your server (creates/updates schema idempotently):
  database/employee_payroll_sqlserver.sql
- Note: the app still expects the same database name as in ConnectionStrings:SqlServer.

5) SQLite (optional, no SSMS)
- Set "UseSqlite": true in appsettings.Development.json and optionally Sqlite:DatabasePath. The app uses EnsureCreated for SQLite instead of SQL Server migrations.
- If you already have a `payroll_dev.db` from an older build, delete it after schema changes so EnsureCreated can recreate tables (SQLite has no automatic migration in this template).

Legacy MySQL
- database/employee_payroll_mysql.sql was from an earlier MySQL setup; the application now targets SQL Server by default.

Run the API
- From the repo root:
  dotnet run --project src/EmployeePayroll.Api/EmployeePayroll.Api.csproj
- Default HTTP URL is shown in the console (launchSettings uses http://localhost:5119).
- On first run, EF applies SQL Server migrations and seeds sample employees and May 2026 payroll if the tables are empty.

Run the Angular UI
- cd src/EmployeePayroll.Web
- npm install
- npx ng serve
- Open http://localhost:4200 (or http://127.0.0.1:4200)
- The SPA expects the API at http://localhost:5119 (see src/environments/environment.ts).

Production notes
- Publish the API (dotnet publish) and point ConnectionStrings:SqlServer at your production SQL instance.
- Build the SPA (ng build) and host static files; add your site origin to CORS in Program.cs if it is not localhost.

Excel export
- Uses ClosedXML. Download name pattern: Payroll_<MonthName>_<Year>.xlsx

Invoice upload & Gemini AI extraction
- Sidebar: Invoices. POST multipart file to /api/invoices/upload (max 10 MB).
- Supported: PDF, DOCX, JPG, JPEG, PNG. Users never type a prompt — extraction uses a hidden backend prompt.
- Google Gemini (gemini-2.5-flash) reads the document and returns JSON fields. If ApiKey is empty, falls back to local text parsing (PdfPig/OpenXml).
- Configure in appsettings.json or user secrets (do not commit real keys):
    "Gemini": { "ApiKey": "YOUR_KEY", "Model": "gemini-2.5-flash" }
  Get a key: https://aistudio.google.com/app/apikey
  Dev: cd src/EmployeePayroll.Api && dotnet user-secrets set "Gemini:ApiKey" "YOUR_KEY"
- PUT /api/invoices/{id} — edit extracted fields after upload (review modal in the UI).
- Files stored under src/EmployeePayroll.Api/InvoiceUploads/ (config: InvoiceUpload:StoragePath).

Monthly tracking
- Sidebar: Monthly tracking. API: /api/monthly-tracking?month=&year=
- Add/edit site work entries (project name, description, status, remarks).

Material tracking
- Sidebar: Material tracking. API: /api/material-tracking?month=&year=
- Add/edit materials (name, qty, unit, rate, total, supplier, received date).

Impress
- Sidebar: Impress. Shows attendance employee names; advance saves to payroll.

Invoice table columns (extracted from PDF/DOCX)
- Invoice No (link opens file), Date, Vendername, SGST, CGST, IGST, Transportcharges, Basic Total, Totalamount
